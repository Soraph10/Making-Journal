## Documentation 

### Friday 13th March
In class we explored and experimented with this in browser software called p5.js. Essentially this is a JavaScript library, it is designed to make coding accessible for designers and beginners. We were given different codes that created various things such as circles, adjusting the size of the shape or changing it overtime. After a few more examples we did a small activity to which we had to recreate the image that was shown. A yellow eclipse with a black background with the the lower part being green. 

It took me a while to get the hang of the different code rules but once I understood what to do it was actually pretty fun making the composition on the slides. In other cases the code replaced the mouse cursor, through removing certain parts of the code  this in turn made the cursor leave afterimages as you dragged it on the canvas. As we progressed further into our lecture we came across DOM (Document Object Model) Elements, which are the structure of a webpage. These elements include buttons, sliders, text inputs. 

Once we had covered this we then had a short intermission, the last section of the lecture covered a quick overview of what Vibe Coding is. So essentially vibe coding is building software by using an LLM or Ai. Which are but is not limited to Gemini, ChatGPT and Claude. You describe what you want the ai to do and paste the results into p5.js editor. I didn't really have an idea I wanted to create so I just described to ChatGPT that I wanted it to create something complex. Which resulted in the creation of a galaxy simulation, sort of although not what I had envisioned a galaxt sketch to look like nonetheless it was something similar to it.

I decided to use my complaints data from experiment 1 and I went with my console or rather gaming to be the complaint I worked with since I know myself to complain everytime the ai messes up in certain games. Or if something is just unfair and resulting in me getting annoyed.

When I used GeminiAI to generate a p5.js sketch for me to look at, I actually learned about the key layout of codes such as the Dropdown Box which was more simpler than I had thought and the Button which had three lines of code I can take note of and use in the future. What I summarised is the you first create the option then position on your canvas and only then you can add onto it. Although the thing I still find to be a bit of struggle to wrap my head around would be the use of words such as "if", "else", "for", "let".
